项目规则：新增或修改类 / 方法 / 属性必须完善 Summary 注释
1. 规则目的
提升代码可读性：让开发者快速理解类 / 方法 / 属性的用途、入参、返回值等核心信息，降低维护成本。
统一注释规范：避免因注释缺失 / 不规范导致的协作效率低下、代码理解偏差问题。
适配 Trae AI IDE 特性：完善的 Summary 可更好地支持 IDE 的智能提示、代码导航等功能。
2. 适用范围
本规则适用于 Trae AI IDE 项目中所有新增的：
类（Class/Interface/Enum 等）
方法 / 函数（包括普通方法、静态方法、异步方法）
属性 / 字段（包括实例属性、静态属性、常量）
注：修改已有类 / 方法 / 属性时，若原 Summary 缺失或不准确，也需补充 / 修正。
